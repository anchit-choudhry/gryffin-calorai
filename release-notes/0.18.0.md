# v0.18.0 Release Notes

**Release Date:** June 22, 2026
**Database Schema:** v20 (frontend IndexedDB) + V22 (backend PostgreSQL via Flyway)
**Status:** Minor Release

---

## Overview

v0.18.0 "Legible Ledger" is a pure frontend UX restructure - no backend or database schema
changes. Three tracks fix information architecture that had outgrown the layout: Progress
is reorganized into four focused tabs with basics-first ordering and a per-tab plain-language
headline; the Dashboard demotes the capture methods to a collapsed disclosure so the diary
and vitals strip lead; and analytics section headers adopt the Field Journal voice with
voiced mono kickers and factual captions on each chart.

---

## Track 1 - Progress Page Restructure

### Frontend - Complete (June 22, 2026)

**`apps/web/src/pages/Progress.tsx`**

- Replaced flat 16-section vertical scroll with a four-tab layout (Nutrition / Body /
  Activity / Plates) using the existing `apps/web/src/components/ui/tabs.tsx` primitive;
  active tab writes `#progress?tab=<name>` so links are deep-linkable and survive refresh
- Restyled tabs as almanac table-of-contents index: JetBrains Mono uppercase labels,
  hairline rule under the row, persimmon active mark, squared corners - NOT the default
  shadcn pill variant
- Basics-first ordering within each tab: Nutrition leads with Daily Calorie Trend before
  macro and micronutrient views; Body leads with raw measurements and composition before
  derived analytics (Weight Projection, Adaptive TDEE, Energy Forecast, Patterns)
- Removed `days === 7 &&` guards that caused Macro Trends and Calorie Distribution to
  silently disappear on the 30-day range; both charts now render at all range settings
- 30-day Macro Trends and Calorie Distribution render as stacked `AreaChart` (legible at
  30 data points); 7-day view retains grouped bar form

**`apps/web/src/components/progress/TabHeadline.tsx`** (new file)

- One plain-language serif sentence at the top of each tab fed by existing computed values
- Nutrition: "Averaged N kcal - N under/over goal; protein on track N/7 days"
- Body: current weight and N-day delta
- Activity: kcal burned and activity session count in the period
- Plates: harvest stamp progress toward next achievement unlock
- Wrapped in `EditorialFrame`; deterministic empty-state ("Not enough records yet for a
  weekly read") - never renders 0 or NaN for unset macros

**`apps/web/src/components/progress/TabHeadline.test.tsx`** (new file, 13 tests)

- All four tab variants, empty-state path, NaN guard, zero-data path

## Track 2 - Dashboard Density and Capture

### Frontend - Complete (June 22, 2026)

**`apps/web/src/pages/Dashboard.tsx`**

- Replaced always-expanded four-method capture grid with a collapsed-by-default disclosure:
  a single "Add to today's log" trigger always visible; expanding via click reveals
  Write / Scan / Speak / Photo; collapses on second click
- Trigger shows `aria-expanded` and `aria-controls` pointing to the panel; `type="button"`
- `AnimatePresence` crossfade (opacity transition, not height animation) respects
  `prefers-reduced-motion` via `shouldReduceMotion` check
- Reordered "Today" section: `DailyVitalsStrip` leads, then diary, then capture
  disclosure, then recently logged / favorites / recurring - optimizes for returning users
  who check status before adding
- `DailyVitalsStrip` moved from sidebar column to top of the lead column so it stacks
  correctly on mobile

**`apps/web/src/state/slices/uiSlice.ts`**

- Added `captureOpen: boolean`, `openCapture()`, `closeCapture()` to `UiSlice` interface
  and `createUiSlice` implementation
- Lifted capture state to the store so the `QuickAddFab` and the inline disclosure open
  the same surface via `App.tsx` `handleAction` - one canonical capture path

## Security Patches (applied this cycle)

- `apps/backend/docker-compose.yml`: backend port bound to `127.0.0.1:8080:8080`
  (loopback only) instead of `0.0.0.0`; prevents unintended exposure on dev machines
- `apps/backend/.../GlobalExceptionHandler.java`: `ConstraintViolationException` handler
  no longer serializes `cv.getPropertyPath()`; returns only `cv.getMessage()`, preventing
  field-path disclosure in validation error responses
- `apps/backend/scripts/refresh-off-products.sh`: `$PARQUET_FILE` SQL-escaped via
  `sed "s/'/''/g"` before interpolation into `COPY` command
- `apps/backend/src/main/resources/application.yml`: `CORS_ALLOWED_ORIGINS` default
  removed; must be set explicitly - no default origin accepted in any environment

## Upgrade Notes

No database migration required - this release makes no IndexedDB or PostgreSQL schema
changes. Pull, run `pnpm install` (if `package.json` changed), and restart the dev
server. Deep links using `#progress?tab=nutrition` (and body / activity / plates) are
now supported.

The four security patches listed above apply to backend configuration and scripts. No
new environment variables are required, but `CORS_ALLOWED_ORIGINS` must now be set
explicitly in `apps/backend/.env` - there is no longer a fallback default.
