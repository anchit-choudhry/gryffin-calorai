# v0.10.0 Release Notes

**Release Date:** June 14, 2026
**Database Schema:** v20 (frontend IndexedDB) + v4 (backend PostgreSQL via Flyway)
**Status:** Major Release

---

## Overview

v0.10.0 adds a nutrition intelligence layer on top of the v0.9.0 design system: an earned-calorie
model that raises your daily target by the calories you burn through activity, a training/rest day
toggle that shifts the macro split between carb-heavy and fat-heavy days, and user-defined custom
macro targets that override the formula when you need precise control. Logging quality improvements
include a photo confirmation form (fill in name and macros before the entry commits), undo-toast on
body measurement deletion, and recipe search inside the Command Palette.

## Nutrition Intelligence

### Frontend - Complete (June 10, 2026)

**`apps/web/src/lib/tdee.ts`**

- `applyPeriodization(base, training)` exported: shifts carb/fat split based on training vs rest
  day; protein target is unchanged; training day adds 30% carbs and removes 10% fat relative to
  the base macro split; rest day removes 20% carbs and adds 10% fat

**`apps/web/src/components/dashboard/DashboardHero.tsx`**

- Earned-calorie model: `earnedGoal = calorieGoal + totalBurned`; the calorie ring, remaining-kcal
  label, and "over by" label all use `earnedGoal` instead of the fixed `calorieGoal`; activity
  calories burned are thus added back to the daily ceiling rather than subtracted from logged intake
- Training/rest day toggle: `aria-pressed` button in the DashboardHero pill row; persisted via
  `toggleTrainingDay(selectedDate)` in `uiSlice`; pressing the button flips the macro targets for
  the selected date via `applyPeriodization()`
- Macro targets now respect both training day periodization and any custom override set in Settings:
  `customMacroGoals?.proteinG ?? periodized.protein` (custom per-gram values take precedence)
- Goal chip in the bottom row now shows the earned goal breakdown when activity is logged:
  `{baseGoal} + {burned} earned`

**`apps/web/src/state/slices/uiSlice.ts`**

- `trainingDays: ISODate[]` state persisted to `localStorage` as `gc_training_days`
- `toggleTrainingDay(date: ISODate)` action: adds date if absent, removes it if present
- `isTrainingDay(date: ISODate): boolean` selector: used by DashboardHero to decide whether to
  apply the training-day macro split

**`apps/web/src/state/slices/settingsSlice.ts`**

- `CustomMacroGoals` interface: `{ proteinG?: number; carbsG?: number; fatG?: number }`
- `customMacroGoals: CustomMacroGoals | null` state persisted to `localStorage` as
  `gc_custom_macros`; loaded on startup via `loadCustomMacroGoals()`
- `setCustomMacroGoals(goals | null)` action: passing `null` clears the override and restores
  formula-derived targets

**`apps/web/src/pages/Settings.tsx`**

- Custom Macro Goals form added to the Goals section: three number inputs (protein / carbs / fat
  in grams); saving writes to `customMacroGoals` in Zustand; a Clear button restores formula mode
- Inputs show the current formula-derived value as a placeholder when no override is set

## Logging Experience

### Frontend - Complete (June 10, 2026)

**`apps/web/src/components/PhotoFoodLogger.tsx`**

- Photo confirmation form: after an image is selected or captured, a draft entry is created with
  empty name, calories, protein, carbs, fat, and meal type; user fills in the form and taps
  "Log Food" to commit - photo is not logged silently without confirmation
- Thumbnail is generated at 120px wide via `OffscreenCanvas` before the form is shown; the full
  image and thumbnail are both stored in the draft and passed to `onPhotoReady` on confirm
- Form fields: Food name (required), Calories (required), Meal type select, and optional Protein /
  Carbs / Fat inputs; all use `EDITORIAL_INPUT_CLS` for consistent styling
- Error feedback via `role="alert"` paragraph: shows "Food name and calories are required" on
  submit without required fields
- Discard button clears the draft and resets the file input; remove photo button discards
  before the form stage
- `PhotoFoodLogger.test.tsx` updated: 14 tests covering confirmation form render, required field
  validation, successful log dispatch, discard flow, and `onPhotoReady` callback

**`apps/web/src/components/BodyMeasurements.tsx`**

- Two-stage delete (confirm / cancel buttons inline in the table row) replaced with immediate
  delete + sonner undo toast: clicking the delete icon fires `deleteBodyMeasurement()` immediately
  and shows a "Measurement removed" toast with an "Undo" action that calls `addBodyMeasurement()`
  with the snapshotted data
- Removes `pendingDeleteId` state; `addBodyMeasurement` now destructured alongside
  `deleteBodyMeasurement` in the component

**`apps/web/src/components/CommandPalette.tsx`**

- Recipe search: when the query is 2+ characters, recipes are filtered by name and description
  (case-insensitive substring match); up to 5 recipe results appear in a "Recipes" category group
  above the standard command categories
- `PaletteRecipe` entry type added alongside `PaletteCommand` and `PaletteFood`; `isRecipe()`
  type guard mirrors the existing `isFood()` guard
- Selecting a recipe result navigates to `#recipes` and closes the palette
- `recipeResults` memo reads `recipes` from Zustand state; no additional DB queries on each
  keystroke
- `CommandPalette.test.tsx` updated: 3 new tests in "recipe search" describe block (match renders
  Recipes category, click navigates, no match hides category)

## Design Chrome Pass

### Frontend - Complete (June 14, 2026)

Six items that remove every remaining stock-library surface from the app, anchoring the
Almanac / Field Journal identity at the shared-primitive level.

**`apps/web/src/App.tsx` - Toasts as margin notes**

- `<Toaster />` restyled entirely via CSS custom properties on `[data-sonner-toaster]`:
  `--normal-bg: var(--paper)`, `--normal-border: var(--rule)`, `--normal-text: var(--ink)`,
  `--border-radius: 0`; squared paper-slip shape with hairline rule border
- Undo action text uses persimmon link color; toast eyebrow label in JetBrains Mono

**`apps/web/src/components/ui/dialog.tsx` - Dialog chrome**

- Container: `rounded-none shadow-none bg-paper text-ink border border-rule` - eliminates generic
  rounded shadowed dialog look
- Four `RuleCorner` SVG ornaments placed at absolute corners (`top-0 left-0`, `rotate-90`,
  `rotate-180`, `-rotate-90`) in `text-rule/50`; 32x32px, L-rule with inner parallel hairline
- `DialogTitle`: changed from generic sans to Spectral serif light:
  `font-serif text-xl font-light text-ink tracking-tight leading-none`

**`apps/web/src/components/ui/skeleton.tsx` - Skeletons as pencil under-drawings**

- Replaced `animate-pulse` shimmer with `skeleton-hatch` keyframe: diagonal `linear-gradient(
  -45deg, ...)` with animated `background-position` from `0 0` to `40px 40px`; 1s linear
  infinite; produces a moving hatching effect that reads as pencil rather than gray wash
- Keyframe defined in `apps/web/src/style.css`

**`apps/web/src/components/dashboard/LogEntry.tsx` - Dot-leader ledger rows**

- Inner content restructured to `flex items-baseline gap-1` row: food name + dot-leader span +
  right-aligned calorie
- Dot-leader: `<span aria-hidden="true" className="flex-1 self-end border-b border-dotted
  border-rule/40 mb-[3px] min-w-[8px]" />` - fills space between name and calories
- Food name capped at `max-w-[60%]` to prevent overflow into leader area

**`apps/web/src/components/recipes/RecipeRow.tsx` - Dot-leader ledger rows**

- Same dot-leader pattern applied to recipe name and calorie value
- Ingredient count moved to its own `<p className="mt-1">` below the leader row to avoid
  layout collision

**`apps/web/src/style.css` - Micro-material pass**

- `::selection { background-color: ... }` - persimmon ink wash at low opacity
- Scrollbar thumb: paper-tone (`--paper`) on webkit scrollbar pseudo-elements
- `accent-color: var(--persimmon)` globally for checkboxes, radios, range inputs

**`apps/web/src/pages/Settings.tsx` - Field Journal colophon**

- About section rebuilt as a typographic colophon: Spectral serif app name at display size;
  bordered dot-leader rows for Edition (v0.10.0), Storage (IndexedDB), Year (2026)
- `APP_VERSION` constant bumped `"0.9.0"` -> `"0.10.0"`

**`apps/web/src/components/PageLoading.tsx` - Almanac page loader**

- `animate-spin` SVG spinner replaced with `<SeasonalFlourish />` SVG ornament (120x24 viewBox,
  horizontal crop mark style) with `animate-pulse` opacity fade
- Loading message text: `font-mono text-[10px] uppercase tracking-[0.3em] text-ink-soft/50`

**`apps/web/src/lib/utils.ts`**

- `SERIF_TITLE_CLS` constant added:
  `"font-serif text-ink font-light tracking-tight lining-nums"` for use on section headings
  wherever Spectral display treatment is needed
