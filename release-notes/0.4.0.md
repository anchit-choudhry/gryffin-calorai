# v0.4.0 Release Notes

**Release Date:** May 23, 2026
**Database Schema:** v15
**Status:** Major Release

---

## Overview

v0.4.0 expands Gryffin Calorai with body measurement UX improvements, projected weight timeline,
recipe import from URL, diet profiles with restriction flags, recurring meal logging, and PWA
support. This release is the last client-only major feature release before v0.5.0 (reminders).

---

## Major Features

### Body Measurements UI Refresh

Inline edit and delete for body measurements with unit toggles.

**What's New:**

- Full edit/delete UI with inline confirmation dialogs on each row
- `updateBodyMeasurement` action in Zustand store and Dexie
- Unit-display toggles (kg/lb, cm/in) in the measurements table
- 21 tests covering all edit/delete/unit-toggle paths

---

### Projected Weight Timeline Card

Shows estimated goal-reach date based on current rate of weight change.

**What's New:**

- `ProjectedWeightCard` component in `src/components/progress/` on Progress page
- `projectedDateForWeightChange()` pure function in `src/lib/tdee.ts`
- Reads body measurement trend and TDEE calorie goal offset to compute estimated date
- Displayed in Progress page Section A2 when a TDEE profile exists

---

### Feature 18: Recipe Import from URL

Removes manual ingredient entry by parsing recipe pages automatically.

**What's New:**

- `src/hooks/useRecipeImport.ts`: `parseRecipeFromHtml()` extracts JSON-LD `@type: Recipe` data
  (name, description, `recipeIngredient[]`); handles `@graph` arrays, malformed JSON, missing fields
- `useRecipeImport()` hook: URL state, fetch via `corsproxy.io`, loading/error states, `clearImport`
- `src/components/recipes/RecipeForm.tsx`: URL import section above the form with URL input,
  Import button, inline error display, and imported-name success indicator
- Ingredient fuzzy-matching via `fuzzyMatchFoodName` on import
- 24 tests in `useRecipeImport.test.ts`, 15 tests in `RecipeForm.test.tsx`

---

### Feature 15: Diet Profiles + Restriction Flags

Declare a diet mode and get automatic warnings when logged foods violate it.

**What's New:**

- `DietPreset` union type: `generic | keto | paleo | vegan | vegetarian | mediterranean |
  high_protein | low_sodium | low_carb` in `src/types/index.ts`
- `RestrictionFlag` union type: `gluten | dairy | nuts | soy | eggs | shellfish | alcohol | pork`
- `checkFoodNameRestrictions(name, flags)` keyword-matching utility; wired into `addFoodLog` to
  toast a warning when a food name matches a restriction keyword
- `DietProfileId` branded type; DB schema v14: `dietProfiles` table with unique index on `userId`
- `src/state/AppState.ts`: `dietProfile` state, `loadDietProfile`, `saveDietProfile` actions
- `src/hooks/useDietProfile.ts`: react-hook-form + zod with defaults from stored profile
- `src/components/DietProfileEditor.tsx`: preset button group + restriction flag toggles
- `computeMacroTargets(calorieGoal, preset)` in `src/lib/tdee.ts` derives P/C/F gram targets;
  consumed by `DashboardHero.tsx` and `MacroStat` progress bars
- New "Diet" section in `src/pages/Settings.tsx` (between Profile and Goals)
- Tests: `DietProfileEditor.test.tsx`, `useDietProfile.test.ts`, updated `AppState.test.ts`

---

### Feature 7: Recurring Meal Logging

Schedule meals that repeat on selected days of the week; prompted to log on app open.

**What's New:**

- `RecurringMeal` entity: inline food snapshot array, `dayMask` (bitmask Mon-Sun), `mealType`,
  `scheduledTime` (HH:MM string), `userId`; `RecurringMealId` branded type
- `getTodayDayIndex()` utility in `src/types/index.ts`
- DB schema v15: `recurringMeals` table with index on `[userId, dayMask]`
- `src/state/AppState.ts`: `recurringMeals` state, `addRecurringMeal`, `deleteRecurringMeal`,
  `updateRecurringMeal`, `checkAndPromptRecurringMeals` actions; prompt runs in `initApp()`
- `src/hooks/useRecurringMealForm.ts`: react-hook-form + zod, `dayMask` default 0b1111111
- `src/components/RecurringMeals.tsx`: list + inline form; integrated into Dashboard

---

### Feature 20: PWA + Service Worker

Installable progressive web app with offline support.

**What's New:**

- `vite-plugin-pwa` integrated in `vite.config.ts`; `registerType: "autoUpdate"`,
  `injectRegister: "script"` (avoids CSP inline-script issues)
- Workbox pre-caches all JS/CSS/HTML/SVG/JPG/WOFF assets; `navigateFallback: "index.html"`;
  `cleanupOutdatedCaches: true`
- Manifest: `display: "standalone"`, `theme_color: "#0A0A0A"`, icons from `scarlet-gryffin.jpg`
  and `favicon.svg`
- `public/_headers`: SW-specific `Cache-Control: no-cache, no-store, must-revalidate` for
  `/sw.js` and `/registerSW.js`; all security headers synced
- Prerequisite satisfied for Feature 17 (Reminders)
