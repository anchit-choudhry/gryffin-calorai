# Changelog - v0.0.9

**Release Date:** May 12, 2026  
**Files Changed:** 42

## Overview

Version 0.0.9 ships two flagship features - manual step tracking and a gamification achievement
system - alongside a major Progress page overhaul, configurable daily goals for water and steps,
new Gemini-powered CI workflows, and a significant expansion of the test suite. The DB schema
advances to v9, AppState gains a 4-state init machine replacing the old boolean `isLoading` flag,
and all utility exports are consolidated in `src/lib/utils.ts`.

---

## Major Features

### 👟 Step Tracking - Feature 5 (`src/components/StepTracker.tsx`)

- New `StepLog` entity tracked in IndexedDB with branded `StepLogId` type and compound
  `[userId+dateLogged]` index (schema v9).
- `StepTracker` component with quick-add presets (2k / 5k / 8k / 10k), custom input, and a
  progress hairline (`sky-500`) filled proportionally against the current goal.
- Inline goal editor: click the step count display to edit the goal in-place; persisted to
  `localStorage` with valid range [1000, 100000].
- Per-session log history list with individual delete. Log entries sorted newest-first.
- `useStepForm` hook: `zodResolver(StepSchema)` validation, optional `stepsOverride` param for
  quick-add bypass, returns `{ form, isLoading, submitStepLog }`.
- `StepSchema` + `StepFormValues` added to `src/forms/schemas.ts`.
- `DAILY_STEP_GOAL = 10000` constant and `StepLogId` branded type added to `src/types/index.ts`.
- Dashboard Section B updated: now renders four cards - WeeklySummary, StreakCard, WaterTracker,
  **StepTracker**.

### 🏆 Gamification Achievement System - Feature 8 (`src/lib/achievements.ts`)

- 20 achievements across 6 categories: **streak** (3/7/14/30-day), **calorie** (on target, full
  day, 3-day streak), **hydration** (first sip, goal hit, 3-day streak), **milestone** (1/10/50/100
  logs), **body** (1/5/10 measurements), **recipe** (1/5 recipes).
- `evaluateAchievements(params: EvaluationParams, alreadyUnlocked: Set<string>): string[]` -
  pure function evaluating all conditions in one pass; never re-awards already-unlocked
  achievements.
- `UserAchievement` entity in IndexedDB (schema v8, compound `[userId+achievementId]` index).
- `checkAndUnlockAchievements()` action in AppState: runs in parallel after every food, water,
  body, and step log mutation; batch-writes newly earned achievements; fires one `sonner` toast
  per unlock.
- `unlockedAchievements: UserAchievement[]` slice added to Zustand store; loaded during
  `fetchInitialData`.
- Progress page Section 07 renders the full achievement grid with locked/unlocked visual states.

### 💧 Configurable Daily Water Goal (`src/components/WaterTracker.tsx`)

- Water goal is now user-configurable via inline editing (same pattern as step goal), persisted
  to `localStorage`, valid range [250, 10000] ml.
- `waterGoalMl` field and `setWaterGoalMl(ml)` action added to AppState.
- Progress bar percentage and display label both derive from `waterGoalMl` instead of the
  previous hard-coded constant.

## Progress Page Overhaul (`src/pages/Progress.tsx`)

Expanded from 3 sections to a **7-section layout** with Recharts throughout:

| Section                | Chart                                                            | Notes        |
|------------------------|------------------------------------------------------------------|--------------|
| 01 Calorie Trend       | `ComposedChart` stacked bars by meal type + goal `ReferenceLine` | 7 or 30 days |
| 02 Macro Breakdown     | `AreaChart` protein / carbs / fat                                | 7-day only   |
| 03 Meal Distribution   | `PieChart` by meal type                                          | 7-day only   |
| 04 Water Intake        | `AreaChart` via `useWaterHistoryData`                            | 7 or 30 days |
| 05 Body Measurements   | `LineChart` weight, body fat %, waist/chest/hips                 | All-time     |
| 06 Measurement History | Table, newest first                                              | All-time     |
| 07 Achievements        | Responsive grid of all 20 achievements, locked/unlocked state    | All-time     |

- **7/30-day toggle** via shadcn/ui `<Tabs>` - switches chart data range for sections 01-04.
- `useReducedMotion()` guards all motion animations for accessibility.

## New Hooks

### `useWaterHistoryData(days: 7 | 30)` (`src/hooks/useWaterHistoryData.ts`)

- Fetches `getAllWaterLogs` from DB; aggregates amounts into daily buckets for the last N days;
  fills missing days with 0.
- Returns `{ labels: string[] (MM-DD), data: number[], isLoading }`.
- Used by Progress page Section 04.

### `useProgressData` Extended (`src/hooks/useProgressData.ts`)

- Now returns `mealTypeData: MealTypeData | null` and `macroData: MacroData | null`.
- Both fields are only populated for the 7-day view; `null` for 30-day, keeping 30-day chart
  rendering paths clean.

### AppState Init Machine (`src/state/AppState.ts`)

- `isLoading: boolean + user: UserProfile | null` replaced with `init: AppInitState` - a
  discriminated union with four states: `idle`, `loading`, `ready`, `error`.
- New store fields: `dailyStepLogs`, `unlockedAchievements`, `waterGoalMl` (localStorage-backed),
  `stepGoal` (localStorage-backed).
- New actions: `fetchAchievements`, `fetchDailyStepLogs`, `addStepLog`, `deleteStepLog`,
  `checkAndUnlockAchievements`, `setWaterGoalMl`, `setStepGoal`.
- `fetchInitialData` now loads step logs and unlocked achievements in parallel with existing data.
- All DB imports aliased to prevent name collisions with action names (e.g.
  `addStepLog as addStepLogToDB`).

### DB Schema v9 (`src/db/dbService.ts`)

| Version | Addition                                                                                       |
|---------|------------------------------------------------------------------------------------------------|
| v8      | `userAchievements` table - `"++id, [userId+achievementId], userId, achievementId, unlockedAt"` |
| v9      | `stepLogs` table - `"++id, [userId+dateLogged], userId, dateLogged"` - **current**             |

New exports: `addUserAchievement`, `getUnlockedAchievements`, `getUnlockedAchievementIds`,
`addStepLog`, `getDailyStepLogs`, `getAllStepLogs`, `deleteStepLog`, `getAllWaterLogs`.

### Vitest Path Alias (`vitest.config.ts`)

- Added `resolve.alias` mapping `@` to `./src` so test files can use the same `@/components/ui/`
  import paths as source files without adjustments.

## Migration Notes

1. Run `pnpm install` - dependency version bumps applied automatically.
2. Run `pnpm lint:fix` to apply formatting updates.
3. Run `pnpm test` - all tests must pass before deploying.
4. Restart dev server with `pnpm dev` - Vite will pick up the new `vitest.config.ts` alias.
5. IndexedDB auto-migrates on first app load via Dexie schema versioning; no manual data
   migration required.
