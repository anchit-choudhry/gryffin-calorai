# Changelog - v0.0.4

**Release Date:** May 5, 2026  
**Files Changed:** 37

## Overview

Version 0.0.4 introduces five major new tracking features (voice food logging, water intake, body
measurements, streaks, and a Suspense-based page loading fallback), hardens security across the
stack, upgrades the Dexie schema to v7, and expands test coverage with a dedicated coverage
reporter.

---

## Major Features & Improvements

### Voice Food Logging (`src/components/VoiceFoodLogger.tsx`, `src/hooks/useVoiceCapture.ts`)

- New `VoiceFoodLogger` component integrates Web Speech API for hands-free food entry.
- `useVoiceCapture` hook wraps `SpeechRecognition` with start/stop control, interim and final
  transcript state, and browser-compatibility guard.
- `sanitizeVoiceTranscript()` added to `src/types/index.ts` - strips non-printable characters and
  enforces a 200-character cap before the transcript reaches the UI or store.
- `fuzzyMatchFoodName()` (Levenshtein distance) resolves spoken food names against the existing item
  list, matching partial or mis-spoken entries.
- Dashboard wired: voice match pre-fills `FoodLogger` in the same inline-card pattern used by the
  barcode flow.
- Tests: `useVoiceCapture.test.ts`, `VoiceFoodLogger.test.tsx`.

### Water Intake Tracker (`src/components/WaterTracker.tsx`, `src/hooks/useWaterForm.ts`)

- `WaterTracker` component renders daily water progress against `DAILY_WATER_GOAL_ML = 2000 ml` with
  add/delete log controls.
- `useWaterForm` hook manages amount input, validation, and Zustand dispatch.
- DB: `waterLogs` table added in **schema v6** (`++id, [userId+dateLogged], userId, dateLogged`).
- New store slice: `dailyWaterLogs[]`, actions `fetchDailyWaterLogs`, `addWaterLog`,
  `deleteWaterLog`.
- Dashboard: rendered alongside `StreakCard` in a responsive 2-column grid.

### Body Measurements (`src/components/BodyMeasurements.tsx`, `src/hooks/useBodyForm.ts`)

- `BodyMeasurements` component on the Progress page for logging weight (kg/lb), body fat (%), waist,
  chest, and hips (cm/in).
- Unit conversion helpers in `src/types/index.ts`: `kgToLb`, `lbToKg`, `cmToIn`, `inToCm`.
  `WeightUnit` and `LengthUnit` branded union types.
- DB: `bodyMeasurements` table added in **schema v7** (
  `++id, [userId+measuredAt], userId, measuredAt`).
- New store slice: `bodyMeasurements[]`, actions `fetchBodyMeasurements`, `addBodyMeasurement`,
  `deleteBodyMeasurement`.
- New branded types: `WaterLogId`, `BodyMeasurementId` with `isWaterLogId` / `isBodyMeasurementId`
  type guards.

### Streak Card (`src/components/StreakCard.tsx`, `src/hooks/useStreaks.ts`)

- `StreakCard` displays the current consecutive-day logging streak and longest streak.
- `computeStreaks()` utility in `src/types/index.ts` accepts a sorted list of ISO date strings and
  returns `{ current, longest }`.
- `useStreaks` hook derives streak state from `dailyLogs` dates without additional DB queries.

### Page Loading / Suspense Fallback (`src/components/PageLoading.tsx`)

- Minimal `PageLoading` spinner component used as the `<Suspense fallback>` in `App.tsx`.
- `App.tsx` converted all three pages to `React.lazy()` and wrapped `renderPage()` in
  `<ErrorBoundary><Suspense fallback={<PageLoading />}>`.
- `PageLoading.test.tsx` added (first component-level test shipped; validates render + ARIA role).

## Security Hardening

### HTTP Security Headers (`vite.config.ts`, `public/_headers`)

- Strict CSP applied in dev/preview server (`vite.config.ts`) and as a static `public/_headers` file
  for Cloudflare/Netlify deployments:
  - `default-src 'self'`; `script-src 'self'`; `style-src 'self' 'unsafe-inline'` (Tailwind JIT
    requirement, documented).
  - `connect-src 'self'` - placeholder comment marks where the barcode food-lookup API origin will
    be added.
  - `frame-ancestors 'none'`; `X-Frame-Options: DENY`; `X-Content-Type-Options: nosniff`.
  - `Permissions-Policy: camera=(self), microphone=(self), geolocation=()` - scopes hardware
    access
    to this origin only.
- `index.html`: `<meta http-equiv="Content-Security-Policy">` tag added for baseline CSP in
  environments that don't inject HTTP headers.

### Database & State Security Fixes

- `clearDatabase()`: now throws in production (`import.meta.env.MODE === "production"`) to prevent
  accidental data wipes.
- `initializeDB()`: schema-conflict auto-recovery guarded to development only; production throws a
  descriptive error requiring manual migration.
- `addFoodItemLog()`: removed the silent `db.delete()` fallback on key-conflict - callers now
  receive the actual error.
- `updateCalorieGoal()`: validates goal is a finite number in `[1, 99_999]` before persisting.
- Ownership checks added: `deleteFoodItem`, `deleteRecipe`, `toggleFavorite`, `updateFoodLog` now
  pass `userId` to DB helpers, preventing cross-user mutations.

## Application Refactoring

### DB Schema (`src/db/dbService.ts`)

- **v6**: Adds `waterLogs` table with compound `[userId+dateLogged]` index.
- **v7**: Adds `bodyMeasurements` table with compound `[userId+measuredAt]` index.
- `console.log` statements removed from DB init path.
- Exported table references (`waterLogs`, `bodyMeasurements`) added after schema definition.

### State (`src/state/AppState.ts`)

- `fetchInitialData` now also pre-fetches today's water logs and all body measurements in parallel.
- Six new actions cover water and body measurement CRUD.
- Error messages use the standard `error instanceof Error ? error.message : "…"` pattern throughout.

### Types (`src/types/index.ts`)

- New branded IDs: `WaterLogId`, `BodyMeasurementId`.
- New type guards: `isWaterLogId`, `isBodyMeasurementId`.
- New sanitizers: `sanitizeBarcodeInput` (moved/consolidated), `sanitizeVoiceTranscript`.
- New utilities: `fuzzyMatchFoodName`, `computeStreaks`, unit converters.
- `DAILY_WATER_GOAL_ML = 2000` constant exported.

### Dashboard (`src/pages/Dashboard.tsx`)

- Three-column logging grid: Food Logger | Barcode Scanner | Voice Log.
- Water Tracker + Streak Card in a responsive 2-column row above the Favorites bar.
- Voice match flow mirrors barcode flow (inline pre-filled `FoodLogger` card with cancel).

### Progress Page (`src/pages/Progress.tsx`)

- `BodyMeasurements` section appended below the chart area with a section divider.
