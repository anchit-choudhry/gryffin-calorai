# v0.3.0 Release Notes

**Release Date:** May 21, 2026  
**Database Schema:** v13  
**Status:** Major Release

---

## Overview

v0.3.0 marks a significant expansion of Gryffin Calorai's core functionality, adding four major
features that address critical gaps in health tracking applications. This release moves the app
from a calorie-counting tool toward a comprehensive health tracking platform with intelligent goal
setting, activity tracking, fasting protocols, and data portability.

---

## Major Features

### ­¤Åā Feature 10: Basic Activity Logging

Log exercises and activities with automatic calorie burn calculation.

**What's New:**

- `ActivityLog` entity with MET-based calorie computation
- `src/lib/metTable.ts`: Static lookup table (~60 common activities with ACSM metabolic equivalents)
- `useActivityForm` hook: react-hook-form + zod validation
- `ActivityLogger` component: searchable activity dropdown, duration input, real-time calorie burn
  display
- `ActivityTracker` summary component for activity overview
- Dashboard integration: activity logs grouped by date with delete-with-undo pattern
- Calorie burn formula: `MET ├Ś weight(kg) ├Ś duration(hours)`
- DB schema v12 ŌåÆ v13: `activityLogs` table with compound index on `[userId+dateLogged]`

**Impact:** Enables net-calorie tracking (intake - burn) critical for weight loss goals.

### ŌÅ▒’ĖÅ Feature 6: Intermittent Fasting

Track fasting windows with real-time progress and browser notifications.

**What's New:**

- `FastingSession` entity: `startTime`, `endTime`, `targetHours`, `userId` (ISO 8601 timestamps)
- `useFastingTimer` hook: elapsed time via `setInterval` + `date-fns differenceInSeconds`
- `FastingTimer` dashboard widget:
  - SVG ring progress indicator (elapsed vs target hours)
  - Preset buttons: 16:8, 18:6, 20:4, OMAD (One Meal A Day)
  - Start/Stop buttons with toast feedback
- Browser Notification API: requests permission on fast start; fires notification when target
  elapsed
- DB schema v13: `fastingSessions` table with compound index on `[userId+startTime]`
- Fasting history persisted to Dexie; accessible in state

**Impact:** Supports growing intermittent fasting trend with customizable fasting protocols.

### ­¤Ä» Feature 13: Onboarding + TDEE / Goal Engine

Personalized calorie targets derived from biometrics and fitness goals.

**What's New:**

- Multi-step onboarding modal (first-launch experience, localStorage flag):
  - Age, sex, height, weight, activity level (sedentary/light/moderate/active/very active)
  - Goal selection (lose/maintain/gain)
  - Unit toggles (kg/lb, cm/in) with automatic value conversion
- `src/lib/tdee.ts`: Mifflin-St Jeor BMR formula ├Ś activity factor = TDEE
  - Goal offsets: -500 kcal/day (lose ~0.5 kg/week), 0 (maintain), +300 kcal/day (gain lean mass)
- `useOnboarding` hook: multi-step navigation, unit conversion, submission
- `OnboardingModal` component: modal UI with step-by-step form
- `OnboardingBanner` component: persistent prompt when no TDEE profile exists
- DB schema v11 ŌåÆ v13: `tdeeProfiles` table; `calorieGoal` exposed via Zustand state
- Settings page (`src/pages/Settings.tsx`): 4 sections
  - **Profile**: `TdeeProfilePanel` - full edit form with live BMR/TDEE/goal preview; unit toggles
    auto-convert
  - **Goals**: `GoalSettings` - inline edit for daily water (ml) and step (steps) targets
  - **Data**: `DataExportPanel` - export & import UI
  - **About**: version info + GitHub link
- Dashboard dynamically uses `calorieGoal` from profile; fallback to 2000 kcal

**Impact:** Personalized goals are table-stakes for engagement. This is a core feature expected
by users of modern health tracking applications.

### ­¤ÆŠ Feature 19: Data Export & Backup

Full data portability: JSON backup + CSV export + import from backup.

**What's New:**

- **JSON Backup Export**:
  - Serializes all Dexie tables to single versioned JSON file (`BACKUP_VERSION = 1`)
  - Includes all food logs, recipes, water logs, step logs, body measurements, activity logs,
    fasting sessions, achievements, TDEE profile
  - Download via Blob URL with timestamp: `gryffin-backup-YYYY-MM-DDTHH:mm:ss.json`

- **CSV ZIP Export**:
  - Uses `fflate zipSync` to produce `gryffin-csv-YYYY-MM-DD.zip`
  - Contains separate CSVs: `foodItems.csv`, `waterLogs.csv`, `stepLogs.csv`,
    `bodyMeasurements.csv`, `activityLogs.csv`, `fastingSessions.csv`
  - Preserves schema structure for Excel/Numbers/Google Sheets import

- **JSON Backup Import**:
  - File picker UI for selecting JSON backup file
  - Schema + version validation
  - `pendingPayload` confirmation dialog (shows merge summary)
  - Upserts all tables via `importData` action (last-write-wins conflict resolution)

- `useDataExport` hook: `downloadJSON` and `downloadCSVZip` actions; shared `isExporting` state
- `useDataImport` hook: file parsing, schema validation, confirmation flow
- `DataExportPanel` component: combined export (JSON + CSV ZIP) and import UI panel
- Adds `BACKUP_VERSION` constant to DB schema v13

**Impact:** Removes data lock-in fear; users can export and switch apps freely. Data portability
is expected by users of modern health applications.

## Security Hardening

Tooling and HTTP header improvements shipped alongside v0.3.0 features.

**HTTP headers (`vite.config.ts`):**

- Added `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload` (HSTS)
- Added `Cross-Origin-Opener-Policy: same-origin` (prevents cross-origin window.opener access)
- Added `Cross-Origin-Embedder-Policy: require-corp` (enables SharedArrayBuffer; blocks unapproved
  cross-origin subresources)
- Added `Cross-Origin-Resource-Policy: same-origin` (prevents this origin's assets from being loaded
  cross-origin without opt-in)

**Package manager (`.npmrc`, `pnpm-workspace.yaml`):**

- Pinned npm registry to `https://registry.npmjs.org/` in `.npmrc` (prevents silent registry
  substitution in compromised CI environments)
- Removed misleading `min-release-age=7` from `.npmrc`; `minimumReleaseAge: 7200` in
  `pnpm-workspace.yaml` is the authoritative setting (5-day buffer)
- Added `verifyStoreIntegrity: true` explicitly to `pnpm-workspace.yaml`

**Scripts (`package.json`):**

- Added `pnpm audit` script (`--audit-level=high`) for CVE scanning

**Node version (`.node-version`):**

- Added `.node-version` file pinning to Node 24; enables auto-switching with nvm/fnm/Volta
