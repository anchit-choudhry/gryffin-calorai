# v0.13.0 Release Notes

**Release Date:** June 17, 2026
**Database Schema:** v20 (frontend IndexedDB) + v4 (backend PostgreSQL via Flyway)
**Status:** Minor Release

---

## Overview

v0.13.0 is a focused design pass. Three systems ship: "Editions" paper theming extends the
Tailwind dark-mode architecture with three new print-register palettes (Sepia / Lamplight /
Large-print); the product tour coachmark is restyled as a field-journal folio page with a
Roman-numeral step counter and dotted leader rule; and letterpress micro-states add a
subtle press-into-paper active animation to chips, buttons, and fasting preset buttons
throughout the app. The WaterTracker also receives a tide-visualization refactor fixing a
react-hooks/immutability lint error.

---

## Editions Theming

### Frontend - Complete (June 17, 2026)

**`apps/web/src/style.css`**

- Three new CSS custom-property token sets scoped to `.edition-sepia`, `.edition-lamplight`,
  and `.edition-large-print` class selectors on `<html>`
- Sepia: parchment-toned backgrounds (`--surface`, `--surface-2`), aged-ink text (`--ink`),
  warm persimmon accent retained; targets a daytime journaling feel
- Lamplight: deep warm dark backgrounds with candlelight amber cast; lower base lightness than
  the default dark mode; intended as a low-blue-light evening register
- Large-print: default-palette base with `--text-scale: 1.2` multiplier driving `1rem` to
  `1.2rem`; no color shift, accessibility-focused; fulfills WCAG large-text threshold

**`apps/web/src/state/slices/uiSlice.ts`**

- `edition` field added to `UiState` with type `"standard" | "lamplight" | "sepia" | "large-print"`
- Persisted to `localStorage` under key `gc_edition`; default `"standard"`
- `setEdition(edition)` action; `useLayoutEffect` in `App.tsx` removes previous edition class
  and adds the new one to `<html>` on every change; coordinates with the existing dark-mode
  class toggle so the two systems do not conflict

**`apps/web/src/pages/Settings.tsx`**

- Two-column Edition picker added to the Appearance section alongside the accent and density
  pickers; four chip buttons with edition name + one-line description

## Field-Journal Tour Coachmark

### Frontend - Complete (June 17, 2026)

**`apps/web/src/components/tour/CoachmarkCard.tsx`**

- Header restyled as a "Field Journal" running title in Spectral serif with a hairline rule
  below; replaces the generic card heading
- Step counter now renders as a Roman numeral folio (`toRoman(stepIndex + 1)`) via the shared
  `toRoman()` utility in `apps/web/src/lib/utils.ts`; matches the letterpress folio style
  used in the Progress page
- Body section uses a dotted leader rule between the coachmark title and description text for
  typographic alignment
- Previous / Next footer uses `Close` wording on the last step; Previous button gets the
  letterpress active state (see below) for consistent interaction feedback
- `ProductTourOverlay` updated: "Quick tour" button renamed "Welcome to Gryffin Calorai" to
  match the field-journal register; step labels updated from "Step N" to "Page N" with Roman
  numeral notation

## Letterpress Micro-States

### Frontend - Complete (June 17, 2026)

**`apps/web/src/style.css`** and component files

- Chips and toggle buttons across Settings (density, accent, edition pickers) receive
  `active:scale-[0.97] active:shadow-[inset_0_1px_2px_oklch(0_0_0/0.18)]` for a press-into-
  paper tactile effect; exit is immediate (no transition on active:) to feel like a physical
  press
- Fasting preset buttons in `FastingTimer.tsx` receive the same treatment
- `CoachmarkCard.tsx` Previous button: same active-state pattern
- No changes to hover or focus styles; letterpress only applies to the active (pressed) state
  to avoid overstyling interactive elements

## WaterTracker Tide Visualization

### Frontend - Complete (June 17, 2026)

**`apps/web/src/components/WaterTracker.tsx`**

- `TIDE_LABELS` constant extracted to module scope; was previously defined inside the render
  function causing a react-hooks/immutability lint error (array recreated on every render)
- Tide mark computation refactored from an imperative loop to `useMemo` + `reduce`; same
  output, stable reference between renders
- Visual layout unchanged; the tide-register strip (intra-day intake events rising toward the
  2000 ml target) still renders identically
