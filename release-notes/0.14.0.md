# v0.14.0 Release Notes

**Release Date:** June 18, 2026
**Database Schema:** v20 (frontend IndexedDB) + v4 (backend PostgreSQL via Flyway)
**Status:** Minor Release

---

## Overview

v0.14.0 completes the second design pass with five almanac-idiom features: the inking moment
animates the most-repeated action (logging food) with SVG stroke-dashoffset choreography; the
fasting timer ring is restyled as a waxing moon disk; a new solar-math library powers a live
almanac strip showing sunrise, sunset, moon phase, and season; the date navigator gets a
paper-shift slide animation; and the broadsheet layout unlocks a newspaper two-column grid for
large screens. Two additional surface upgrades ship alongside: a navigation running head with
serif folio and ledger-line form fields. No new dependencies, no DB schema changes.

## Fasting Timer as Lunar Cycle

### Frontend - Complete (June 18, 2026)

**`apps/web/src/components/FastingTimer.tsx`**

- The progress ring is replaced with a moon-disk SVG: a full dark circle as the base, a
  persimmon ellipse that grows from a sliver to a full circle as the fasting window progresses,
  clipped to the right half via `clipPath` so the growth reads as waxing
- A moon-phase icon (crescent or full) sits inside the disk; its phase is derived from the
  current time and the fasting start time, not the calendar moon, giving the user visual
  feedback proportional to their actual fasting progress
- SVG coordinates are literal values (not variables) to ensure the clip path and disk align
  pixel-exactly regardless of React re-renders

## Today's Almanac Panel

### Frontend - Complete (June 18, 2026)

**`apps/web/src/lib/solar.ts`**

- `getDayOfYear(date)`: day-of-year (1-365/366); used for the phenology wheel tick position
- `getSeason(date)`: returns `"Spring" | "Summer" | "Autumn" | "Winter"` based on
  Northern Hemisphere astronomical boundaries
- `getMoonPhase(date)`: Julian Day Number algorithm returning `{ phase: number; name: string }`
  where `phase` is 0-1 (0 = new moon, 0.5 = full moon); astronomically correct
- `getSunTimes(lat, lng, date)`: NOAA solar-angle algorithm returning `{ sunrise, sunset }`
  as `Date` objects; accurate to within ~1 minute for latitudes -65 to +65

**`apps/web/src/components/dashboard/AlmanacPanel.tsx`**

- Four-cell editorial strip on the Dashboard: sunrise time, sunset time, moon phase name +
  icon, season name + day-of-year; designed as a newspaper almanac column entry
- Location form with a lat/lng label field; location is optional and never leaves the device;
  a privacy note ("Location stays on your device") appears below the field
- Lazy-loaded with `React.lazy`; skeleton shown while loading; renders nothing when no
  location is set and the user has dismissed the prompt

**`apps/web/src/state/slices/uiSlice.ts`**

- `almanacLocation?: { lat: number; lng: number; label: string }` persisted under
  `gc_almanac_loc` (JSON)
- `broadsheet: boolean` persisted under `gc_broadsheet`; toggles the desktop two-column
  layout; default `false`
