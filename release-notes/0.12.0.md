# v0.12.0 Release Notes

**Release Date:** June 17, 2026
**Database Schema:** v20 (frontend IndexedDB) + v4 (backend PostgreSQL via Flyway)
**Status:** Major Release

---

## Overview

v0.12.0 ships ten features across three clusters: Almanac artifacts (printable journal page,
phenology wheel, food specimen sheets, micronutrient heatmap), social sharing without a backend
(shareable PNG cards, QR recipe exchange, privacy dashboard), and home-screen convenience (PWA
share-target registration, quick-log deep links). All features are offline-first; nothing leaves
the device unless the user explicitly shares. A 30-day weight forecast line completes the
adaptive analytics suite started in v0.11.0.

## Shareable PNG Cards

### Frontend - Complete (June 17, 2026)

**`apps/web/src/lib/shareCard.ts`**

- `renderStreakCard(streak, goal, period)`: draws a 1200x630 canvas using the editorial
  palette - paper background, double hairline border, Spectral serif headline, streak count

**`apps/web/src/components/StreakCard.tsx`**

- "Share" button wired to `useShareCard(() => renderStreakCard(...), "streak.png")`; replaced

**`apps/web/src/components/WeeklyHarvestModal.tsx`**

- Share button similarly wired to `useShareCard(() => renderHarvestCard(...), "harvest.png")`

## QR Recipe Sharing

### Frontend - Complete (June 17, 2026)

**`apps/web/src/lib/recipeShare.ts`**

- `encodeSharePayload(recipe)`: serializes a recipe to JSON, compresses with `fflate` deflate,
  base64url-encodes into a URL fragment - zero-server sharing with no backend required
- `decodeSharePayload(encoded)`: decompresses and validates the fragment; returns the recipe
  or `undefined` if the payload is malformed or truncated
- `importSharedRecipe(encoded)`: decodes and writes the recipe to IndexedDB via `addRecipe`;
  called from `App.tsx` when the URL hash matches `#recipes?share=`

**`apps/web/src/components/recipes/RecipeShareSheet.tsx`**

- Bottom-sheet overlay with the encoded share URL, a QR code rendered client-side via
  `qrcode` library, and a "Copy link" button; launched from `RecipeRow` via a share icon
- 12 tests covering QR render, copy-to-clipboard success toast, and the URL shape

## 30-Day Weight Forecast

### Frontend - Complete (June 17, 2026)

**`apps/web/src/lib/adaptiveTdee.ts`**

- `computeWeightForecast(measurements, days?)`: linear regression over the most recent 30
  days of body measurements; projects weight forward `days` (default 30) using the fitted
  slope; returns `{ date, label, projected }[]` with dates incremented by `shiftISODate`
  to avoid DST skew
- Filters to measurements within 30 days of the most recent entry before fitting, so old
  outliers do not distort the projection
- Renders as a dashed forecast line on the weight trajectory chart in the Progress page

## Micronutrient RDA Heatmap

### Frontend - Complete (June 17, 2026)

**`apps/web/src/components/progress/MicronutrientHeatmap.tsx`**

- 7-day heatmap grid showing daily intake of five key micronutrients (vitamin C, calcium,
  iron, fiber, sodium) as a percentage of the personalized RDA from `getPersonalizedRDA()`
- Cell colors: OKLCH lightness gradient from paper (0% RDA) to persimmon (100%+ RDA);
  cells clamp at 120% to avoid misleading over-saturation
- Tooltips show the raw intake value and unit alongside the percentage; sourced from the
  `useProgressData` hook's 7-day food log slice

## Upgrade Notes

No breaking changes or schema migrations required. All new features are additive:

- Phenology Wheel and Micronutrient Heatmap appear automatically in the Progress page when
  body measurement and food log data exist
- Food Specimen Sheet opens from Pantry entries and the Today's Log section
- Privacy Dashboard accessible in Settings under a new "Privacy" section
- Shareable PNG cards: Share buttons appear on StreakCard and the Weekly Harvest modal
- QR recipe sharing: share icon added to RecipeRow; import auto-runs on `#recipes?share=`
  URL on app open
- Printable Almanac: navigate to `#print` or use the Print entry in the navigation menu
- PWA share target active after the app is installed to the home screen; no user action needed
- Quick-log deep links: `#log?water=250` etc. usable from iOS Shortcuts or Android Tasker
