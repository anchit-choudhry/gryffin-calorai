## Overview

Version 0.0.8 represents a significant UI/UX overhaul and architectural enhancement. This release
introduces the shadcn/ui component library for consistent, accessible primitives, establishes a
comprehensive editorial design system with oklch color palette and @fontsource typography, and
modernizes form validation across all inputs using react-hook-form and zod. The Dashboard is
completely restructured with a 5-section editorial layout, motion animations are applied throughout,
and the codebase gains improved type safety and utility functions.

---

## Major Features & Improvements

### 🎨 Design System & UI Components

#### **shadcn/ui Component Library Integration** (`src/components/ui/`)

- Integrates shadcn/ui primitives: Button, Card, Dialog, Form, Input, Label, Tabs, Tooltip with
  Radix UI foundations.
- Includes variant files for Button (`button-variants.ts`) and Tabs (`tabs-variants.ts`) for
  flexible styling.
- All overlays use Dialog with built-in focus trap and Esc-key closing for accessibility.
- Form components provide field-level error display via `<FormMessage />`.

#### **Editorial Design System** (`src/style.css`, theme tokens)

- Establishes oklch color palette with editorial semantics: `paper`, `paper-muted`, `ink`,
  `ink-soft`, `rule`, `persimmon`, `persimmon-soft`.
- Introduces @fontsource-variable typography stack: Fraunces (display serif), Public Sans (sans),
  JetBrains Mono (monospace).
- Design token system with radius scales (`--radius-sm`, `--radius-md`, `--radius-lg`,
  `--radius-xl`).
- `style.css` completely rewritten with Tailwind 4 theme integration and custom variants.

#### **Dashboard Refactoring** (`src/pages/Dashboard.tsx`, `src/components/dashboard/`)

- New 5-section editorial layout: Hero section, Weekly summary, Pantry (favorites), Add to Log
  controls, Today's Log list.
- New dashboard subcomponents:
  - `DashboardHero.tsx` - greeting and high-level overview
  - `EditorialFrame.tsx` - responsive grid layout wrapper
  - `LogEntry.tsx` - individual food log entry card with edit/delete
  - `SectionHeader.tsx` - section dividers with typography
  - `MacroStat.tsx` - macro nutrient stat display
  - `DateKicker.tsx` - date navigation component

#### **Toast Notifications** (`sonner` integration)

- Integrated Sonner for lightweight, accessible toast notifications.
- `<Toaster />` component added to App.tsx for app-wide notification support.

#### **Navigation Enhancement** (`src/App.tsx`)

- Redesigned navigation bar with editorial typography and persimmon accent color.
- Improved hash routing with `normalizeHash()` utility and `ValidHash` type system for type-safe
  route handling.

### 🧪 Form Validation & Type Safety

#### **Zod Schema Validation** (`src/forms/schemas.ts`)

- Centralized form validation with comprehensive zod schemas for all inputs:
  - Food item logging (name, calories, macros, meal type)
  - Recipe management (name, ingredients, instructions)
  - Water intake (amount, date)
  - Body measurements (weight, body fat, dimensions, units)
- Field-level error handling with react-hook-form integration.

#### **Enhanced Form Hooks**

- All form hooks (`useFoodForm`, `useRecipeForm`, `useWaterForm`, `useBodyForm`) refactored to use
  react-hook-form + zod.
- Improved validation, error messaging, and async field handling.
- Better handling of form state, submission, and reset logic.

### ✨ Animation & Motion

#### **Motion Animation System** (`src/lib/motionVariants.ts`)

- New `motionVariants.ts` exports reusable animation patterns: `pageVariants`, `sectionVariants` for
  page transitions and section stagger effects.
- `motion` 12 (`motion/react`) integrated for layout animations on log list with stagger timing.
- Respects `useReducedMotion()` for accessibility (reduced motion preferences).

### 🛠 Infrastructure & Utilities

#### **Utility Library** (`src/lib/utils.ts`)

- `cn()` - Tailwind class merging with clsx + tailwind-merge
- `normalizeHash()` - Type-safe hash route validation
- `mapDbError()` - Dexie error mapping for user-friendly messages
- Exported as core helper module for form and styling operations.

#### **Component Reorganization**

- New folder structure: `src/components/dashboard/` (dashboard-specific components),
  `src/components/ui/` (shadcn primitives).
- BarcodeScanner refactored with new test coverage.
- Voice and Water logging components updated for editorial design consistency.
