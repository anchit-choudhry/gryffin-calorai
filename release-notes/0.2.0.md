## Overview

Version 0.2.0 introduces interactive product tour and keyboard accessibility features, enabling new
users to explore the app with guided coachmarks and empowering power users with keyboard shortcuts
for all major actions. The release includes a spotlight animation system, step-by-step navigation
UI, global keyboard command registry, comprehensive test coverage, and enhanced motion variants.
This update significantly improves onboarding UX and keyboard accessibility without adding database
schema changes or breaking dependencies.

## Major Features & Improvements

### 🎯 Interactive Product Tour System

#### **ProductTourOverlay Component** (`src/components/tour/ProductTourOverlay.tsx`)

- New root tour component managing spotlight animation, step navigation, and progress tracking.
- Renders spotlight background with animated rect highlighting current target element.
- Provides Next/Prev/Skip button controls with keyboard support (ArrowRight, ArrowLeft, Esc).
- Displays progress counter (e.g., "Step 2 of 8") and step-specific coachmark card.
- Motion animations via `motion/react` with `spotlightVariants` for smooth entrance/exit.
- Prefers-reduced-motion aware with fallback to instant animations.
- Full test coverage: 200+ lines of test cases covering spotlight behavior, step transitions,
  keyboard navigation, and edge cases.

#### **CoachmarkCard Component** (`src/components/tour/CoachmarkCard.tsx`)

- Floating card component displaying tour step title, body text, and navigation buttons.
- Dynamic positioning with auto-placement (top/bottom/left/right) and intelligent fallback to
  prevent off-screen rendering.
- Animated arrow indicator pointing to the highlighted target element.
- Keyboard event handling: Tab navigation through buttons, Enter to confirm, Escape to exit.
- Motion animations with staggered button appearance and arrow bounce effects via
  `coachmarkVariants`.
- Responsive design with proper spacing on mobile devices.
- Full test coverage: 300+ lines including rendering, positioning, keyboard events, and arrow
  animations.

#### **Tour Steps Configuration** (`src/components/tour/tourSteps.ts`)

- Centralized configuration of 8 guided tour steps across 3 pages: Dashboard, Recipes, Progress.
- TourStep type with: id, page, targetId, title, body, isFirst flag, and optional placement hint.
- Steps guide users through core features: Dashboard Hero (calorie ring), Food Logging (
  text/barcode/voice), Hydration & Movement tracking, Today's Log view, Recipe creation, Recipe
  browsing, Progress charts, and Achievements.
- Full test coverage: Configuration validation and completeness.

#### **useSpotlightRect Hook** (`src/components/tour/useSpotlightRect.ts`)

- Custom hook calculating spotlight bounds around target element with smooth animations.
- Handles scroll offsets, window resize events, and dynamic element repositioning.
- Returns spotlight dimensions (x, y, width, height) and loading state.
- Supports `prefers-reduced-motion` media query for accessibility.
- Zero-dependency hook using Intersection Observer for efficient visibility tracking.
- Properly cleans up event listeners and observers on unmount.

#### **Tour State Management** (`src/state/AppState.ts`)

- New store fields: `tourStarted`, `currentTourStep`, `tourCompleted`.
- New actions: `startTour()`, `nextTourStep()`, `prevTourStep()`, `skipTour()`.
- Tour progress synced to localStorage: `gryffin_tourState`.
- Automatically skips tour if user already completed it (one-time onboarding experience).
- Full test coverage in `AppState.tour.test.ts`: 130+ lines covering initialization, step
  advancement, completion, and localStorage persistence.

### ⌨️ Keyboard Shortcuts System

#### **KeyboardShortcutsOverlay Component** (`src/components/KeyboardShortcutsOverlay.tsx`)

- New modal overlay displaying all available keyboard commands in a scrollable list.
- Organized shortcuts by category: Navigation, Food Logging, Tour, Theme, and Utility.
- Clean, readable layout with key badge styling and command descriptions.
- Accessible via global keyboard shortcut (? or Shift+?).
- Close button and Escape key to dismiss.
- Responsive design: full-screen modal on mobile, centered dialog on desktop.

#### **useKeyboardShortcuts Hook** (`src/hooks/useKeyboardShortcuts.ts`)

- Global keyboard event listener managing all application shortcuts.
- Supported commands:
  - **Navigation:** ArrowRight/ArrowLeft (tour prev/next), #/Ctrl+K (quick navigate to
    dashboard/recipes/progress)
  - **Food Logging:** Ctrl+L or Cmd+L (focus food logger input)
  - **Tour:** T (start tour), Escape (exit tour)
  - **Theme:** D (toggle dark mode)
  - **Shortcuts Help:** ? or Shift+? (show/hide shortcuts overlay)
- Prevents event propagation to avoid conflicts with form inputs and browser defaults.
- Properly cleans up event listeners on unmount.
- Full test coverage: Hook registration, command dispatch, event handling.

### 🎨 Enhanced Motion & Animation System

#### **Motion Variants Extended** (`src/lib/motionVariants.ts`)

- New `coachmarkVariants` with entry animation (scale + fade), exit animation, and staggered button
  children.
- New `spotlightVariants` with fade in/out for smooth spotlight entrance/exit.
- New `arrowVariants` with subtle bounce effect on coachmark arrow indicator.
- All variants respect `prefers-reduced-motion` media query.
- Updated `pageVariants` and `sectionVariants` with refined timing (180ms duration).
- Full test coverage: 90+ lines validating all motion sequences and reduced-motion behavior.

### 🎭 Skeleton Loading Component

#### **Skeleton Component** (`src/components/ui/skeleton.tsx`)

- New shadcn/ui primitive for displaying skeleton loaders (placeholder UI while content loads).
- `DashboardSkeleton` export: 4-section layout matching Dashboard content structure.
- `PageSkeleton` export: generic skeleton for any page with flexible sections.
- Animated pulse effect with Tailwind animation `animate-pulse`.
- Used in Dashboard page Suspense fallback for smooth loading UX.
- Full accessibility: `aria-busy="true"` and semantic structure.

## Type System & Architecture

### **Tour Types** (`src/components/tour/tourSteps.ts`)

- `TourStep` interface: id, page, targetId, title, body, isFirst?, placement?
- `TourPage` type: `"dashboard" | "recipes" | "progress"`
- `TourPlacement` type: `"below" | "above" | "auto"`

### **AppState Tour Extensions**

- `tourStarted: boolean` - Track if tour has started
- `currentTourStep: number` - Current step index (0-based)
- `tourCompleted: boolean` - Whether user has completed tour
- Actions: `startTour()`, `nextTourStep()`, `prevTourStep()`, `skipTour()`

### **Keyboard Command Registry**

- Centralized command definitions in `useKeyboardShortcuts` hook
- Easy to extend with new commands following existing patterns
- Type-safe callback handling with consistent event propagation rules

## Known Limitations & Future Work

1. **Tour completion tracking** - Currently localStorage-only; would benefit from backend
   persistence in multi-device scenarios
2. **Tour customization** - Tour steps are hardcoded; future versions could support user-skipped
   steps or role-based tours
3. **Keyboard command documentation** - Integrated into app (? key); could expand with printable
   cheat sheet or help page
4. **Accessibility refinements** - Full WCAG 2.1 AAA audit pending; current implementation covers
   ARIA labels and reduced-motion support

## Performance Considerations

- **Bundle Impact** - Tour components are lazy-loaded with main app; no separate code split needed
- **Memory Usage** - useSpotlightRect uses Intersection Observer for efficient visibility tracking
- **Animation Performance** - All motion animations respect prefers-reduced-motion; motion 12
  provides GPU acceleration
- **Event Listeners** - All keyboard and resize listeners properly cleaned up on unmount to prevent
  memory leaks

## How to Update

To update to v0.2.0:

1. Run `pnpm install` to ensure dependencies are current.
2. Run `pnpm lint:fix` to apply formatting fixes.
3. Run `pnpm test` to validate all changes and check coverage (especially tour tests).
4. Restart the development server with `pnpm dev`.
5. Open the app in your browser and press `?` to view available keyboard shortcuts.
6. Start the product tour by clicking the tour icon or pressing `t`.
